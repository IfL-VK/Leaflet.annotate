Introduction

This plugin extends the options for the LeafletJS Standard API elements Marker, CircleMarker, Popup, ImageOverlay and GeoJSON so you can publish these elements in HTML semantically annotated with machine readable geo markup. While there is quite some effort already around the development of new geospatial data catalogs (which are awesome) this plugin turns the focus onto geographic web maps acknowleding that, in fact, some maps can be considered data catalogs themselves. They are just not that well represented in HTML so that other programs can extract and re-use the information people aggregate and visualize with them. So to make some of the most important geographic elements to be published by the year 2025 on the web in HTML

  • identifiable and addressable for others,
  • searchable for users through search engines,
  • re-usable and interoperable for application developers,
  • real collections of linked data and therewith,
  • potentially multilingual and
  • self-contained and reproducible

Therefore we wrote this small script, integrating freeing you of caring much about the necessary syntax details.

This is (as LeafletJS itself) BSD Licensed open source software. Your feedback or contributions are very welcome. Please use this issue tracker to report any issues you experienced with it.

Installation

View the source of this content.

The source code released contains two different components.

  • The core file is the semantic markup generating Leaflet Plugin in

    Leaflet.annotate.Microdata.js

    After loading this script you can start annotating Marker, CircleMarker, Popup, ImageOverlay and GeoJSON elements.
  • An add-on which operates on the semantic markup generated by the core component is in

    Leaflet.annotate.Viewer.js

    Using this add-on you can add a text based search control to your map allowing readers of it to magically list, search and focus all your annotated web map elements.

Furthemore, to assist the creation of valid semantic markup. Currently you can choose between two JavaScript objects, one in english and one in german language, which helps you to map user inpuot to valid Schema.org itemtype options. We ship this to help you validate user input you pass on to the core generating the markup for you.

Usage

Automatic annnotation

Annotation happens automatically whenever you add a Marker, CircleMarker, ImageOverlay or Popup to your L.map and pass them a valid code>itemtype value as an option to Leaflets Standard API:

L.marker(coordinates, {
  title: 'Madison Square Garden', itemtype: 'CivicStructure'
})

Valid itemtype values are currently all the names of Schema.org terms which allow for at least one Place as one of their properties. Regarding the schema.org type you use you might be required to explicitly specify the desired geoprop option

L.circleMarker(coordinates, { title: „...“ itemtype: 'CreativeWork', geoprop: 'locationCreated' })

For example when mapping CreativeWorks you can express a geographical dimension for the location it was created (locationCreated) or for the location it depicts (contentLocation) using the respective property names.

Here is another, more comple example, annotating an ImageOverlay: L.imageOverlay(imageUrl, imageBounds, { "itemtype": "CreativeWork", "geoprop": "contentLocation", "publisher": "The University of Texas", "source": "https://www.lib.utexas.edu/maps/new_jersey.html", "created": "Thu Jan 01 1920 01:00:00 GMT+0100 (CET)" "title": "Automobile Blue Book", }).addTo(map)

On a GeoJSON element annotation happens whenever you call the .annotate() method on the GeoJSON layer and pass a valid code>itemtype value as an option: 

L.geoJson(geojsonData, {
  title: „Uganda“ itemtype: 'Country',
  sameAs: 'https://www.wikidata.org/wiki/Q1036'
}).annotate()

This will markup the GeoJSON document as a map overlay representing the area for the Republic of Uganda. In Wikidata the URI for the concept/entity representing this country is https://www.wikidata.org/wiki/Q1036.
